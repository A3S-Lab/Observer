# Linux 4.19 ARM64 Observer Hotfix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a standalone AArch64 `a3s-observer-collector` whose embedded legacy BPF programs load on Linux 4.19 without BTF or backward jumps.

**Architecture:** Compile a verifier-constrained C probe object with pinned LLVM, validate its ELF structure in Node.js, and pass the verified object to the existing Rust collector build through `A3S_LEGACY_BPF_OBJECT`. Keep userspace event enrichment and forwarding unchanged.

**Tech Stack:** BPF C, LLVM/Clang, Node.js, Rust/Aya, cargo-zigbuild, Docker
